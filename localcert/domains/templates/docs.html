Here's some things to try

  • Generate a Let's Encrypt certificate for your domain
    • You can use your certificate for testing HTTPS on localhost
    • You can create additional subdomains as you'd like, each with their own certificate
    • You can create a wildcard certificate for your domain
  • Edit your /etc/hosts to setup private routing
  • Configure your router, personal DNS server, or corporate DNS server to route these domains

Keep in mind the following limits of this service:

  • You can only set TXT records for _acme-challenge
    • This will allow Let's Encrypt and similar Certificate Authorities to validate your ownership
  • You can't add A, AAAA, MX, NS, CNAME, or any other record types on the global DNS
    • You can set these entries on any private DNS server you operate, but these won't be assessible from the Internet

Intended use cases:

  • Testing HTTPS on localhost
  • Avoid the use of self-signed certificates or custom Certificate Authorities on internal networks
  • Minimizing costs through free tier offering
